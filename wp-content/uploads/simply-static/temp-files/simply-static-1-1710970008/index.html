W3C 采用新版《行为准则》

March 18, 2024 | 国际新闻

2024年3月18日，W3C 宣布采用新版《行为准则》。 W3C 行为准则（简称《准则》）清晰地定义了在社区中的适当及不当行为，推广高水准的职业行为实践。 该《准则》的目的是：

  •        * 确保每位参与者都得到公平对待与尊重
  •        * 定义可接受的行为标准
  •        * 助力打造良好的组织形象
  •        * 指出不可接受的行为
  •        * 为参与者提供解决问题的指南

新版《准则》取代了先前发布的2020版《道德和职业行为准则》。

W3C 提案推荐标准：IMSC 假设渲染模型

February 29, 2024 | 国际新闻,标准进展

2024年2月29日，W3C 时序文本工作组发布 IMSC 假设渲染模型（IMSC Hypothetical Render Model）提案推荐标准。该规范定义了一个假设渲染模型（Hypothetical Render Model 简称 HRM），约束符合 IMSC 规范中指定的文本配置文件的文档呈现复杂度。

该模型并非是针对实现的处理需求的规范。例如，虽然该模型定义字形缓存是为了对如何减少字形绘制操作的数量进行建模，但它既不要求实现这样的缓存，也不要求对可用于产生文本输出的子像素字形定位和抗锯齿字形渲染进行建模。此外，该模型并不限制可读性的复杂度。

[注] W3C 提案推荐标准（Proposed Recommendation 简称 PR）是已具备足够条件可成为 W3C 正式推荐标准的规范。文档已经过开发者、W3C 工作小组以及相关方的审查。在其成为正式推荐标准前，W3C 会员将对规范进行最终审阅。

征集参考实现：DPub-ARIA 1.1、DPub-AAM 1.1

February 27, 2024 | 国际新闻,WAI,标准进展

2024年2月27日，W3C 可访问富互联应用工作组发布以下两份候选推荐标准，并征集参考实现：

数字出版 WAI-ARIA 模块1.1：要让使用辅助技术的用户找到浏览 Web 内容的方式，就需要嵌入有关 Web 文档结构划分的语义元数据。这对于长格式文档的结构划分尤其重要，并且与嵌入有关 Web 应用程序部件和辅助技术行为的语义元数据相一致。该规范定义了一组 WAI-ARIA 角色，专门用于帮助辅助技术用户浏览此类长格式文档。

数字出版无障碍 API 映射 1.1 定义了用户代理如何将数字出版 WAI-ARIA 模块 [dpub-aria-1.1]标记映射至平台的无障碍 API。它面向负责无障碍可访问性的用户代理开发人员，让他们能够支持为数字出版生成的无障碍内容。

[注] W3C 候选推荐标准（Candidate Recommendation 简称 CR）意味着对规范的技术设计基本完成，现开始进行广泛审阅并征集实现和测试（Call for Implementation）。

欢迎2024年4月27日前通过 GitHub issues反馈对上述规范文档的意见与建议。

W3C 国际化标准负责人变更及未来工作计划

February 02, 2024 | 国际新闻

2024年2月2日消息：W3C 团队成员 Richard Ishida 和薛富侨于2023年底前共同负责 W3C 的国际化标准工作。Richard 自2024年1月1日起转为半退休状态，薛富侨接任 W3C 国际化标准全球负责人一职，开始全面负责所有国际化相关的标准化活动。Richard 则主要关注语言支持的相关工作。

W3C 国际化标准计划（Internationalization 简称 I18N）通过确保 HTML、CSS、SVG 等技术支持世界各地的语言文字的排版需求，使网页和电子书的 UI 和行为（如文字的书写方向，姓名、地址、时区、货币、日期的格式等）符合用户的期望。I18N 是一种设计范式，致力于实现让 Web 打破地域障碍。I18N 是一架桥梁，它消除数字鸿沟，推动数字经济与个人发展，促进文化理解与包容。I18N 是实现 Web 服务于所有人（即 “Web for All”）的支柱之一。

更多内容，参见 W3C 发布的博客文章（英文、中文）。

征集参考实现：可验证凭证数据模型2.0

February 01, 2024 | 国际新闻,标准进展

2024年2月1日，W3C 可验证凭证工作组发布可验证凭证数据模型 2.0候选推荐标准，并征集参考实现。凭证（credentials）是我们日常生活的重要部分：驾照用于证明我们能够驾驶机动车辆；学位证书表明我们的受教育程度；政府签发的护照则用于证明持有者的身份。该规范提供了一种机制，以加密安全、尊重隐私和机器可验证的方式在 Web 上表现此类凭证。工作组积极寻求针对该规范的反馈以及实现，欢迎2024年4月1日前通过 Github 提交反馈。

此外，小组还发布了可验证凭证规范目录。这是一份汇总已知的可验证凭证相关规范的非正式目录，包括由全球标准组织、社区团体、开源项目以及个人发布的规范。欢迎通过 GitHub提交对该文档的反馈。

[注] W3C 候选推荐标准（Candidate Recommendation 简称 CR）意味着对规范的技术设计基本完成，现开始进行广泛审阅并征集实现和测试（Call for Implementation）。

January 31, 2024 | 国际新闻,HTML/Web平台,标准进展

2024年1月31日，W3C MiniApps 工作组发布 MiniApp Addressing 规范的首个公开工作草案。该规范定义了 MiniApp Addressing 的方式，包含基于 URI 规范之上的 MiniApp URI 的具体语法组件的规格说明，以及解引用 MiniApp URI 的过程。实现本规范使得用户代理（UA）能够定位 MiniApp 的资源。

首个公开工作草案：可访问富互联应用（WAI-ARIA）1.3版

January 23, 2024 | 国际新闻,WAI,标准进展

2024年1月23日，W3C 可访问富互联应用工作组发布可访问富互联应用（WAI-ARIA）1.3版规范的首个公开工作草案。该规范提供了角色、状态和属性的本体，这些本体定义了可访问的用户界面元素，可用于改进 Web 内容和应用程序的可访问性和互操作性。此版本添加了新特性，以提高与辅助技术的互操作性。自上一版以来的主要变化包括：添加新的角色（suggestion、comment、mark），新的属性（aria-description、aria-braillelabel、aria-brailleroledescription），更新 aria-details 以允许多个 IDrefs。参阅更新详情。

欢迎于2024年2月21日前反馈对该文档的意见与建议。访问 WAI-ARIA Overview 进一步获取更多资源。

W3C 候选推荐标准：数据目录词汇表（DCAT）第三版

January 18, 2024 | 国际新闻,标准进展,数据

2024年1月18日，W3C 数据集交换工作组发布数据目录词汇表（DCAT）第三版规范（Data Catalog Vocabulary (DCAT) - Version 3）候选推荐标准。DCAT 定义了一个 RDF 词汇表，目的是促进 Web 上发布的数据目录之间的互操作性。

本文档对 DCAT 第二版规范 [VOCAB-DCAT-2] 进行了重大修订，以整合在先前的词汇表开发期间无法考虑的用例、需求和社区经验。该版本扩展了 DCAT 标准使其更符合社区实践，同时支持多种数据描述和数据集交换方法。

欢迎于2024年2月15日前，通过 GitHub 反馈对该文档的意见与建议。
